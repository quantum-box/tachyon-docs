# Agent APIチャットイベントの表示不具合修正

## 概要
`/v1beta/tn_01hjryxysgey07h5jz5wagqj0m/ai/agent/chat` 画面でエージェント実行ログの一部イベント（例: `ask`）が表示・処理されない問題を解消し、サーバーから送信される全イベントを安定してレンダリングできるようにする。

## 背景・目的
- 2025-10-14 時点で、Agent API 実行時にサーバーから `ask` などのイベントが届いてもUI側で型定義がなく描画されないとの報告がある。
- ユーザーがエージェントとの対話に必要な追加入力が提供できず、フロントエンドの操作性を大きく損なっている。
- 全イベントを正しく扱えるようにし、UI での処理待ちやエラー表示を改善するのが目的。

## 詳細仕様

### 機能要件 📝
1. SSE/ストリーム経由で受信する全てのイベントタイプ（`say`/`assistant`、`ask`、`tool_call`、`tool_call_args`、`tool_result`、`usage` など）を網羅した型定義を整備する。
2. `ask` イベント到達時にユーザーへ質問内容を提示し、選択肢を押下すると即座に返信が送信されるインタラクションを提供する。
3. 未対応イベントが来た場合はデフォルトのフォールバック表示を行い、コンソールエラーとならないようにする。
4. 既存のチャット履歴レンダリング（バブル表示・タイムスタンプ等）の体裁は維持する。

### 非機能要件 📝
- 既存のローディングやエラー表示のUXを損なわないこと。
- Biome/Lint チェックおよび `mise run check` に合格すること。
- 型追加によりバンドルサイズが極端に増えないよう配慮する。

### コンテキスト別の責務
- `apps/tachyon`（フロントエンド）: SSE ハンドリング、UI 表示、追加入力の送信ロジック補強。
- バックエンド変更は不要な想定だが、必要に応じて Mock/型の整合性確認を行う。

### 仕様のYAML定義
現時点で必要な構造化仕様はなし。イベント定義は TypeScript の型ガードで表現する。

## 実装方針 📝
- 現状のイベントハンドリングコードを調査し、サーバーからのイベントpayloadを確認する。
- TypeScript の Discriminated Union を用いてイベント型を整理し、未対応タイプを補完する。
- `ask` イベント用の UI コンポーネントを追加し、回答送信時に既存API呼び出しロジックを再利用する。
- フォールバックとして未知イベントをログ出力しつつ UI に簡易表示するようにする。
- 仕様確定後、Playwright MCP による動作確認を行い、スクリーンショットを残す。
