# エージェントAPIの並列ツール実行機能の実装

## 概要

エージェントAPIにおいて、複数のツールを並列実行できる機能を実装します。現在は順次実行されているツール呼び出しを、並列実行可能にすることで、エージェントの処理速度を大幅に向上させます。

## 背景・目的

### 現状の課題
- 現在のエージェントAPIは、ツールを1つずつ順次実行しており、複数のツール呼び出しが必要な場合に時間がかかる
- 例：3つの独立したAPIを呼び出す場合、各APIが2秒かかるとすると合計6秒必要
- ユーザー体験の観点から、応答時間の短縮が求められている

### 期待される成果
- 独立したツール呼び出しを並列実行することで、処理時間を最大で1/nに短縮（n=並列実行数）
- エージェントの応答性向上によるユーザー体験の改善
- システム全体のスループット向上

## 詳細仕様

### 機能要件

1. **並列実行の判定**
   - エージェントが複数のツールを呼び出す際、依存関係のないツールを自動的に識別
   - 並列実行可能なツールグループを形成

2. **並列実行の実装**
   - 識別されたツールグループを並列で実行
   - 各ツールの実行結果を非同期で収集
   - すべての結果が揃ったら、エージェントに制御を戻す

3. **ストリーミングレスポンス**
   - 並列実行中のツールの進捗をリアルタイムでクライアントに送信
   - 各ツールの開始・完了をSSEで通知

4. **エラーハンドリング**
   - 並列実行中の個別エラーを適切に処理
   - 一部のツールが失敗しても、他のツールの実行を継続

### 非機能要件

- **パフォーマンス**: 並列実行により、独立したツール呼び出しの総実行時間を最も遅いツールの実行時間まで短縮
- **スケーラビリティ**: 同時実行数の上限を設定可能（デフォルト: 5）
- **後方互換性**: 既存の順次実行APIとの互換性を維持

## 実装方針

### アーキテクチャ設計

```plaintext
[Agent] 
   ↓ ツール呼び出し要求
[Tool Executor]
   ↓ 依存関係分析
[Parallel Executor]
   ├→ [Tool A] ←→ [External API A]
   ├→ [Tool B] ←→ [External API B]
   └→ [Tool C] ←→ [External API C]
   ↓ 結果集約
[Agent]
   ↓ 後続処理
[Response Stream]
```

### 技術選定

**バックエンド (Rust)**
- `tokio::spawn` for 並列タスク実行
- `tokio::sync::mpsc` for 結果収集
- `futures::stream` for ストリーミング処理

**フロントエンド (TypeScript/React)**
- 既存の `useAgentStream` フックを拡張
- 並列実行の視覚的表現を追加

### データ構造の変更

```typescript
// 新しいチャンクタイプを追加
interface AgentChunk {
  // 既存のフィールド
  tool_call?: { name: string };
  tool_call_args?: string;
  tool_result?: string;
  
  // 新規追加
  parallel_group_id?: string;  // 並列実行グループの識別子
  parallel_status?: 'started' | 'completed' | 'failed';
}
```

## リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 並列実行によるリソース枯渇 | 高 | 同時実行数の上限設定、リソースモニタリング |
| 依存関係の誤判定 | 中 | 保守的な判定ロジック、明示的な依存関係宣言 |
| 既存APIとの非互換性 | 低 | フィーチャーフラグによる段階的ロールアウト |
