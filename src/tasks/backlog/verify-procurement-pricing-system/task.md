# 調達・価格管理システムの動作確認

## 概要

実装済みの調達システムと価格管理システムを使用して、Agent APIが正しく価格設定され、課金フローが期待通りに動作することを確認する包括的なテストタスクです。

## 背景・目的

### 解決したい課題
- 調達価格（ProcurementPrice）が正しく設定されているか不明
- Agent API実行時の価格計算が正確に行われているか未検証
- 課金フローが実際のAgent実行で正しく動作するか未確認
- エンドツーエンドでのシステム統合が機能しているか不明

### 期待される成果
- Agent APIの価格設定が正しく機能することの確認
- 課金フローが期待通りに動作することの検証
- システム統合の問題点の発見と修正
- 本番環境での安定稼働に向けた信頼性の確保

## 詳細仕様

### 動作確認フロー

#### エンドツーエンドの確認フロー
1. **クレジットチャージ**
   - Stripe経由でのクレジット購入をシミュレート
   - クレジット残高への反映確認

2. **調達原価の確認**
   - LLM APIの調達コスト（ProcurementPrice）確認
   - Agent実行の調達原価設定確認

3. **Agent APIの価格設定**
   - 販売価格（CatalogPrice）の設定
   - マージンの確認（販売価格 - 調達原価）

4. **Agent API使用**
   - 実際のAgent実行
   - リアルタイムコスト計算

5. **クレジット消費**
   - 設定した金額分のクレジット消費確認
   - 残高の更新確認

6. **履歴確認**
   - トランザクション履歴の確認
   - 実行コスト詳細の確認

### 機能要件

#### 1. クレジットシステムの動作確認
- クレジットチャージのフロー確認
- 残高管理の正確性
- トランザクション記録の完全性

#### 2. 調達原価管理の確認
- LLM API（Claude, GPT等）の調達コスト設定
- Agent実行の基本コスト設定
- ツール使用コストの設定

#### 3. 価格設定と販売の確認
- カタログ価格の設定機能
- 価格の有効期間管理
- マージン計算の確認

#### 4. 実行時の課金フロー
- 事前の課金可能性チェック
- 実行中のコスト計算
- 実行後のクレジット消費

#### 5. 履歴とレポーティング
- トランザクション履歴の検索・表示
- 実行コスト詳細の表示
- 使用統計の確認

### 非機能要件
- 各APIエンドポイントの応答時間が適切であること
- エラーメッセージが適切に返されること
- ログが正しく出力されること

## 実装方針

### テスト環境の準備
1. ローカル開発環境でのシステム起動
2. テスト用テナントの準備（シードデータ使用）
3. 必要なデータベースの初期化

### 確認手順
1. **データベース確認**
   - 価格データの存在確認
   - クレジット残高の確認
   - 既存トランザクションの確認

2. **API動作確認**
   - GraphQL経由での価格情報取得
   - Agent Chat UIでの実行
   - Playwrightでの自動テスト

3. **結果検証**
   - クレジット消費の確認
   - トランザクション履歴の確認
   - エラーケースの検証
