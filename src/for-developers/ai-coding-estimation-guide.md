# AIコーディング時間見積もりガイド

AIコーディングツール（Claude Code/Cursor）を使用した開発における精密な時間見積もり手法について説明します。

## 概要

従来の手作業によるコーディングと比較して、AIコーディングツールは劇的な生産性向上をもたらしています。本ガイドは実測データに基づいた見積もり手法を提供します。

## 簡易見積もり式

```
見積もり時間 = (ファイル数 × 1分) × 複雑度係数 × 環境係数
```

## ファイルタイプ別基準時間

### 基準時間一覧

| ファイルタイプ | 基準時間 | 1行あたり時間 | 特記事項 |
|--------------|---------|-------------|----------|
| Rust モデル | 1分 | 0.75秒/行 | 型安全性でエラー少 |
| Rust API/ロジック | 1分 | 0.6秒/行 | 効率的 |
| Next.js/React ページ | 1.5分 | 0.82秒/行 | UI要素多めで時間増 |
| TypeScript ユーティリティ | 0.5分 | 0.67秒/行 | 標準的 |
| 設定ファイル | 0.5分 | 1.5秒/行 | 正確性重要 |
| SQL/Migration | 0.5分 | 0.75秒/行 | 標準的 |

### 詳細説明

#### Rustファイル
- **モデル定義**: 型定義やstructの実装は型安全性により高速に実装可能
- **API/ロジック**: Clean Architectureパターンに従った実装で効率的
- **エラー処理**: Result型による明確なエラーハンドリング

#### TypeScript/React
- **コンポーネント**: UI要素の多さにより基準時間が長め
- **ユーティリティ**: 純粋な関数は迅速に実装可能
- **型定義**: TypeScriptの型システムによりバグを削減

#### 設定・インフラ
- **設定ファイル**: 正確性が重要なため慎重な実装が必要
- **SQL/Migration**: スキーマ定義は標準的な時間で実装

## 実例計算

### 例1: 新規API開発

```
条件:
- ファイル数: 5個（Model 2個、API 3個）
- 複雑度: 中（1.5）
- 環境: 新規プロジェクト（1.0）、慣れた技術（1.0）、明確要件（1.0）

計算:
見積もり = 5 × 1分 × 1.5 × 1.0 = 7.5分
```

### 例2: 複雑な認証システム

```
条件:
- ファイル数: 8個
- 複雑度: やや高（1.8）
- 環境: 既存プロジェクト（1.3）、慣れた技術（1.0）、明確要件（1.0）

計算:
見積もり = 8 × 1分 × 1.8 × 1.3 = 18.7分
```

### 例3: レガシーシステムのリファクタリング

```
条件:
- ファイル数: 15個
- 複雑度: 高（2.5）
- 環境: レガシー改修（2.0）、やや不慣れ（1.5）、やや曖昧（1.8）

計算:
見積もり = 15 × 1分 × 2.5 × 2.0 × 1.5 × 1.8 = 202.5分 ≈ 3.4時間
```

## 従来手法との比較

### 効率化率

| 作業規模 | 従来見積もり | AI見積もり | 効率化率 |
|---------|------------|-----------|---------|
| 小規模（~10ファイル） | 1-2日 | 10-20分 | 24-48倍 |
| 中規模（~30ファイル） | 1週間 | 30-60分 | 84-168倍 |
| 大規模（~100ファイル） | 2-3週間 | 2-4時間 | 60-120倍 |

### 効率化の要因

1. **自動コード生成**: ボイラープレートコードの自動生成
2. **エラー検出**: リアルタイムでの構文・型エラー検出
3. **パターン認識**: 既存コードパターンの学習と適用
4. **並列処理**: 複数ファイルの同時編集

## 使用時の注意点

### 1. 初期フェーズは保守的に見積もる
- 環境設定エラーが発生しやすい
- プロジェクト構造の理解に時間がかかる
- 初回は1.5倍程度のバッファを見込む

### 2. 学習効果を考慮
- 2フェーズ目以降は効率が向上
- 同じパターンの実装は高速化
- チーム全体のスキル向上も考慮

### 3. 要件の明確化が最重要
- 曖昧さは1.8倍の時間増
- 事前の仕様検討に時間をかける
- プロトタイピングの活用

### 4. 技術スタック統一を推奨
- 切り替えコストの削減
- AIツールの学習効果を最大化
- チーム間での知識共有が容易

## 推奨ワークフロー

### 1. 事前準備（5-10分）
- ファイル数とタイプを洗い出し
- 既存コードの構造を理解
- 必要な仕様を整理

### 2. 見積もり（2-3分）
- 複雑度レベルを評価（1-5）
- 環境係数を算出
- 式に当てはめて計算

### 3. 実装（見積もり時間）
- AIツールでの実装
- リアルタイムでのエラー修正
- コードレビュー

### 4. バッファ考慮
- ±30%のバッファを考慮
- 不確実性が高い場合は+50%
- 初回実装は+100%も検討

## ベストプラクティス

### 効率を最大化するために

1. **明確な仕様書を用意**
   - AIへの指示が明確になる
   - 手戻りを最小化

2. **小さな単位で実装**
   - 1回の実装を10ファイル以下に
   - エラーの影響範囲を限定

3. **既存パターンを活用**
   - プロジェクトの規約を事前に理解
   - サンプルコードを準備

4. **継続的な検証**
   - 実装後すぐにテスト
   - CI/CDパイプラインの活用

### 避けるべきこと

1. **大規模な一括変更**
   - エラーの原因特定が困難
   - ロールバックが複雑

2. **曖昧な指示**
   - AIの誤解を招く
   - 修正に時間がかかる

3. **セキュリティの軽視**
   - 自動生成されたコードも要レビュー
   - セキュリティホールに注意

## まとめ

AIコーディングツールは適切に使用すれば、従来の24-168倍の効率化が可能です。ただし、要件の明確化と適切な見積もりが成功の鍵となります。本ガイドの手法を活用し、プロジェクトの特性に応じた調整を行うことで、より精度の高い見積もりが可能になります。
