# プロパティタイプ一覧

データベースサービスでは、様々なタイプのデータを格納するためのプロパティタイプを提供しています。
各プロパティタイプは、データの構造と制約を定義します。

## サポートされているプロパティタイプ

| プロパティタイプ | 説明 | 詳細ドキュメント |
|--------------|------|--------------|
| String | 文字列を格納するための基本タイプ | - |
| Integer | 整数値を格納するためのタイプ | - |
| Html | HTML形式のテキストを格納するためのタイプ ※非推奨（Markdownへ移行予定） | - |
| Markdown | Markdownテキストをプレーンな文字列として格納するタイプ | - |
| Relation | 他のデータベースとの関連を表すためのタイプ | - |
| Select | 単一の選択肢を格納するためのタイプ | - |
| MultiSelect | 複数の選択肢を格納するためのタイプ | - |
| Id | 一意の識別子を格納するためのタイプ | - |
| Location | 位置情報（緯度・経度）を格納するためのタイプ | [詳細](./location.md) |

## プロパティタイプの選択基準

データベース設計時には、格納するデータの性質に合わせて適切なプロパティタイプを選択することが重要です。

- **String**: テキストデータやシンプルな文字列
- **Integer**: 数値計算や順序付けが必要なデータ
- **Html**: 既存データの後方互換用途。新規コンテンツにはMarkdownを利用する想定
- **Markdown**: Markdown記法で表現されたリッチテキストコンテンツ
- **Relation**: 他のデータベースのレコードとの関連付け
- **Select**: 定義済みの選択肢から1つを選ぶ場合
- **MultiSelect**: 定義済みの選択肢から複数を選ぶ場合
- **Id**: 自動生成または手動指定の一意識別子
- **Location**: 地理的な位置情報のデータ

## 使用方法の基本

プロパティタイプを使用するには、データベースにプロパティを追加する際にタイプを指定します。
各タイプには固有の入力形式と検証ルールがあります。詳細は各タイプのドキュメントを参照してください。
