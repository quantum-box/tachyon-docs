# User Insights

## Overview

User Insights automatically summarize long-lived preferences and recurring topics from an operator's recent chat messages. Saved Memory remains user-controlled, while Insights are generated by Tachyon to help the agent adapt tone and disambiguate future requests.

## Feature highlights

- Weekly/On-demand regeneration of insights by clustering recent user messages (max 200 within 180 days).
- LLM-generated JSON summaries with confidence level, supporting message IDs, and keywords.
- Stored in `tachyon_apps_llms.user_insights` and injected into the agent system prompt alongside Saved Memory.
- REST API for listing/regenerating insights (`/v1/agent/insights`, `/v1/agent/insights/regenerate`).
- UI tab on `/ai/memory` showing cards, confidence badges, date ranges, and regenerate CTA.

## Generation pipeline

1. Fetch recent user messages via `UserInsightMessageRepository` (role=USER, text only, within 180 days, limit 200).
2. Sample up to 80 messages for prompt context.
3. Call `LLMCompletionDomainService` with strict JSON instructions:

```json
{
  "insights": [
    {
      "summary": "User always requests TypeScript examples first",
      "confidence": "HIGH",
      "message_ids": ["me_01...", "..."],
      "keywords": ["typescript", "frontend"]
    }
  ]
}
```

4. Validate output: require >=2 supporting messages, truncate summary (400 chars), derive keywords if missing.
5. Replace all existing insights for the user in a transaction.

Each user/tenant pair is guarded by an in-memory lock to prevent concurrent regeneration via REST or cron jobs.

## REST API

| Method | Endpoint                           | Description                | Auth action              |

## Agent prompt injection

`ExecuteAgent` now fetches both Saved Memories and User Insights. The default system prompt gains a `USER INSIGHTS` section:

```
====

USER INSIGHTS

The following insights were derived automatically from the user's prior conversations. Use them to adapt tone, clarify ambiguous requests, or provide tailored explanations.

1. (High confidence, Feb 2025 – Mar 2025) User has asked multiple questions about Rust async streams and wants concise code samples.
```

## UI

- `/v1beta/[tenant_id]/ai/memory` uses a new tabbed layout (`Saved Memory` / `User Insights`).
- The insights tab shows cards with confidence badges, date range, keywords, and message counts.
- "Regenerate insights" button invokes a server action and revalidates the page.
- Localization keys added under `memory.page.tabs` and `memory.insights`.

## RBAC

- Added actions `llms:ListUserInsights` and `llms:ManageUserInsights`, attached to `ai_memory` policy.

## Scheduled regeneration

- AWS Lambda `lambda-user-insights` runs the cron job. The EventBridge schedule defaults to every Monday (`var.user_insight_cron_schedule`) and can be adjusted via Terraform.
- Lambda environment variables:
  - `USER_INSIGHT_CRON_LOOKBACK_DAYS` (default: `7`)
  - `USER_INSIGHT_CRON_MAX_USERS` (default: `200`)
- Local execution: `mise run cron-user-insights` reuses the same binary and logs stats to STDOUT.
