# Colleca — プロダクト要求仕様書 (PRD)

## 1. 背景・目的

* 課題: 楽天ROOMは楽天市場限定。ユーザーはAmazonや海外ECも混在して"お気に入り棚"を共有したいが、現状はモール横断で簡単に作れない。

* サービス名: Colleca (コレカ) — Collection + Marketplace の造語。

* ビジョン: "どのECでも見える商品を、Pinterestライクにコレクションしてシェアし、成果をプラットフォームとクリエイターで安全に分配する"。

## 2. 目標 (OKR 例)

| 期間 | Objective | KR1 | KR2 |
|------|-----------|-----|-----|
| Q3   | β版リリース & 1,000 MAU | 100 コレクション/日 | 初月GMV ¥1,000,000 |

## 3. ターゲットユーザー & ペルソナ

* Creator Nana (23): SNSフォロワー5kの大学生。プチプラファッションを紹介して副収入を得たい。スマホ完結UXが必須。

* Blogger Ken (31): 技術ブログPV 月10万。書籍やガジェットリンクを整理し、記事外でも収益化したい。

## 4. 主なユースケース & ユーザーストーリー

1. Nana は Amazon と楽天の商品URLを貼り付け、1分で「春コーデ」コレクションを作成 → SNSで拡散。

2. Ken は API で自ブログの推薦書籍リストを自動同期 → 月末に収益レポートと振込通知を受け取る。

## 5. プロダクトスコープ (MVP)

### 5.1 コア機能

* マルチECコレクション作成
  * 商品URL貼り付けで自動情報取得
  * コレクション名・説明・カバー画像設定
  * 商品並べ替え・コメント追加

* コレクション共有
  * 一意のURL生成
  * SNSシェアボタン
  * 埋め込みウィジェット

* アフィリエイト連携
  * 各ECのアフィリエイトID自動付与
  * 収益トラッキング
  * 月次レポート

* ユーザー管理
  * SNS認証
  * プロフィール設定
  * 収益振込先登録

### 5.2 技術要件

* フロントエンド
  * Next.js + TypeScript
  * Tailwind CSS
  * Responsive Design (Mobile First)

* バックエンド
  * Rust (axum)
  * GraphQL API
  * MySQL (TiDB)

* インフラ
  * AWS (ECS, Aurora, S3)
  * CI/CD (GitHub Actions)

### 5.3 非機能要件

* パフォーマンス
  * ページロード < 2秒
  * API応答 < 500ms

* セキュリティ
  * HTTPS対応
  * CSRF対策
  * XSS対策

* スケーラビリティ
  * 10,000 MAUまで対応

## 7. 成功指標

* ユーザー数: 登録ユーザー、MAU、DAU
* エンゲージメント: 作成コレクション数、平均滞在時間
* 収益: GMV、アフィリエイト収益、手数料収入

## 8. リスクと対策

* ECサイトのAPI制限: 複数のスクレイピング手法を用意
* アフィリエイト規約変更: 主要ECの規約を定期確認
* 競合参入: 独自のUX/UIで差別化
