# Library 多言語対応（i18n）

## 概要

Library アプリは日本語と英語の2言語に対応しています。ユーザーはブラウザの言語設定に基づいて自動的に言語が選択されるか、手動で言語を切り替えることができます。

## 対応言語

| 言語コード | 言語名 | デフォルト |
|-----------|--------|----------|
| `ja` | 日本語 | ✅ |
| `en` | English | - |

## アーキテクチャ

### ファイル構成

```
apps/library/src/
├── lib/i18n/
│   ├── constants.ts              # Cookie 名、有効期限などの定数
│   ├── translations.ts           # 基本翻訳辞書（Locale 型、共通翻訳）
│   ├── auth-translations.ts      # 認証画面の翻訳
│   ├── dashboard-translations.ts # ダッシュボードの翻訳
│   ├── v1beta-translations.ts    # v1beta 配下の翻訳
│   └── useTranslation.ts         # 翻訳取得フック
│
├── app/i18n/
│   ├── i18n-provider.tsx         # React Context Provider
│   ├── detect-locale.ts          # 言語検出ロジック
│   └── get-dictionary.ts         # 辞書取得ユーティリティ
│
└── components/
    └── language-switcher.tsx     # 言語切り替えコンポーネント
```

### 技術選定

| 技術 | 選定理由 |
|------|---------|
| React Context | アプリ全体で翻訳を共有するため |
| Cookie ベースの言語保持 | SSR でも言語検出可能、URL パスを変えずに済む |
| TypeScript 辞書 | 型安全性、IDE 補完、コンパイル時検証 |

## 使い方

### Server Component での使用

```tsx
import { detectLocale } from '@/app/i18n/detect-locale'
import { getDictionary } from '@/app/i18n/get-dictionary'

export default async function Page() {
  const locale = await detectLocale()
  const dictionary = getDictionary(locale)

  return <h1>{dictionary.dashboard.title}</h1>
}
```

### Client Component での使用

```tsx
'use client'

import { useTranslation } from '@/lib/i18n/useTranslation'

export function MyComponent() {
  const { t, locale, changeLocale } = useTranslation()

  return (
    <div>
      <h1>{t.dashboard.title}</h1>
      <button onClick={() => changeLocale('en')}>
        Switch to English
      </button>
    </div>
  )
}
```

### 言語切り替え UI

`LanguageSwitcher` コンポーネントを使用すると、ドロップダウンメニューで言語を切り替えられます。

```tsx
import { LanguageSwitcher } from '@/components/language-switcher'

export function Header() {
  return (
    <header>
      <LanguageSwitcher />
    </header>
  )
}
```

## 言語検出の優先順位

1. **Cookie に保存された言語** - ユーザーが明示的に選択した言語
2. **Accept-Language ヘッダー** - ブラウザの言語設定
3. **デフォルト言語 (ja)** - 上記が検出できない場合

## 翻訳辞書の追加方法

### 新しい翻訳キーの追加

1. 該当する翻訳ファイル（例: `v1beta-translations.ts`）を開く
2. `en` と `ja` の両方に翻訳を追加

```typescript
export const v1betaTranslations = {
  en: {
    myFeature: {
      title: 'My Feature',
      description: 'This is my feature',
    },
  },
  ja: {
    myFeature: {
      title: '機能名',
      description: 'これは機能の説明です',
    },
  },
}
```

3. `translations.ts` の `Dictionary` 型に新しいキーを追加（型安全性のため）

## 対応ページ一覧

### 認証画面
- サインイン (`/sign_in`)
- サインアップ (`/sign_up`)
- サインアウト (`/sign_out`)
- パスワードリセット (`/forgot-password`, `/reset-password`)
- メール認証 (`/verify-email/otp`)

### ダッシュボード
- ホーム (`/`)

### v1beta ページ
- 組織一覧・詳細 (`/v1beta/[org]`)
- リポジトリ一覧・詳細 (`/v1beta/[org]/[repo]`)
- 設定ページ (`/v1beta/[org]/[repo]/settings`)
- データ詳細 (`/v1beta/[org]/[repo]/data/[dataId]`)
- 組織作成 (`/v1beta/organization/new`)
- リポジトリ作成 (`/v1beta/[org]/databases/new`)

## LP（ランディングページ）について

LP は独自の翻訳システム（コンポーネントローカルの `copy` オブジェクト）を使用しており、i18n システムとは統合されていません。LP の言語切り替えはクエリパラメータ `?lang=en` で行います。
